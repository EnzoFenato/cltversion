Gerenciamento de Configuração de Software

O gerenciamento de configurações de software se baseia em uma análise do software tanto pela parte externa dele(interface), quanto pelo interior(código), para analisar e verificar se está pronto para lançamento dentro de sua data de entrega sem erros e sem necessitar de futuras manutenções, dentre essas analises se destacam:

Controle de Versão

O Controle de versão se baseia numa verificação no código para lidar e rastrear mudanças no código para seus gerenciamentos e com base nessas verificações o Controle de versão mostra diversas ferramentas para ajudar os desenvolvedores no teste do código do software

Controle de Mudanças

O controle de mudanças se baseia numa investigação rápida para verificar se todos os métodos ou procedimentos foram aplicados ao verificar as futuras mudanças e prevenir se as mudanças aplicadas não tragarão futuras manutenções dentro do software

Integração Contínua

A Integração Contínua é uma prática feita pelos desenvolvedores do código para por dentro de um de algo todas as alterações feitas dentro do código para sua mudança, um exemplo disso é o próproa GitHub Desktop, onde toda alteração feita depois de um "Push" é marcada dentro do código para saberem o que mudaram para realizar a manutenção
